📖 How it works

deck_setup

Each card entry maps its name to an array where the first element is the quantity, followed by any number of tags:

"Card Name": [qty, "Tag1", "Tag2"]

You can add as many custom tags as you like — tags are used to define combo patterns and clump rules.

Reserved tags

  • Starter Opens a combo by itself. Typically a Normal Summon that enables a one-card combo (e.g. Aluber, Branded Fusion).
  • Extender Continues or extends a combo already started. Can be a hand trap with an effect, a Called by the Grave, etc.
  • Tech A situational tech card (hand traps, board breakers). Used by the min_techs metric columns.

A card can hold both Starter and Extender tags simultaneously — that's perfectly valid.

NS tag

Use NS as a custom tag to mark Normal Summon cards. It has no special engine behavior — it's just a tag like any other.

combo_patterns

A list of tag/card arrays. Each array represents a distinct combo that can be executed from the hand.

[
  ["Starter"],          // any single Starter
  ["TagA", "TagB"],     // combo needing both tags
  ["SpecificCard"]      // exact card name also works
]

clump_rules

Pairs of cards/tags that are bad together. A clump triggers when both match in the same hand. Use !Tag to negate — the clump triggers only if that tag is absent.

[
  ["CardA", "CardB"],      // bad if both in hand
  ["CardC", "!Extender"]   // bad if CardC + no Extender
]

Metrics explained

✅ Success Rate

  • 0 techs → probability of seeing 1+ plays
  • 1 tech → probability of seeing 1+ Tech cards (ignores plays)
  • 2 techs → probability of seeing 2+ Tech cards (ignores plays)

🤝 Good Hand Rate

  • 0 techs → 1+ plays
  • 1 tech → 1+ techs and 1+ plays
  • 2 techs → 2+ techs and 1+ plays

🎯 Resilience Rate

  • 0 techs → 2+ plays
  • 1 tech → 1+ techs and 2+ plays
  • 2 techs → 2+ techs and 2+ plays

💀 Dead Hand Rate

Hand with 0 plays or any clump triggered.
